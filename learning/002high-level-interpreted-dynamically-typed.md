# JavaScript: High-Level, Interpreted, Dynamically Typed

When someone says:

> **JavaScript is a high-level, interpreted, dynamically typed programming language.**

It means:

## 1. High-Level

**Definition:**
A language that is **easy for humans to read and write**. It hides complex hardware details.

**Example:**

```javascript
let sum = 10 + 20;
```

**Remember:**  
🧑 High-Level → Easy for humans

---

## 2. Interpreted

**Definition:**
JavaScript code is **executed by a JavaScript engine** (like V8).

**Example:**

```javascript
console.log("Hello");
```

**Note:** Modern JavaScript engines use **JIT (Just-In-Time) compilation** to improve performance.

**Remember:**  
🔄 Interpreted → Executed by a JavaScript engine

---

## 3. Dynamically Typed

**Definition:**
A variable's **type is decided at runtime** and can change.

**Example:**

```javascript
let x = 10; // Number
x = "Hello"; // String
```

**Remember:**  
🎭 Dynamically Typed → Variable type can change

---

# Summary

| Term                  | Meaning                             |
| --------------------- | ----------------------------------- |
| **High-Level**        | Easy for humans to read and write   |
| **Interpreted**       | Executed by a JavaScript engine     |
| **Dynamically Typed** | Variable type can change at runtime |

---

## Interview Answer

> JavaScript is a **high-level** language because it is easy to read and write, **interpreted** because it runs using a JavaScript engine, and **dynamically typed** because variable types are determined at runtime and can change.
